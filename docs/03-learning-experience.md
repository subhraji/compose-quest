# The Learning Experience

## Philosophy

Learning Compose should feel like **a guided adventure**, not reading a manual. Three principles:

1. **Always oriented.** The learner never wonders "where am I / what's next / how far to go." The roadmap answers this at a glance.
2. **Understand, don't memorize.** Every idea is delivered four ways — plain, child-level, analogy, and an interactive visual — so it *clicks* before any code.
3. **Do immediately.** Reading is followed within seconds by doing: a live playground and a quick quiz, with a mentor on standby.

## The lesson loop (repeatable, predictable)

Every authored lesson follows the same rhythm so learners build a comfortable habit:

```
 ┌──────────────────────────────────────────────────────────────┐
 │  1. Objective      "Here's the one thing you'll be able to do" │
 │  2. Understand     Explain it · Like I'm 10 · Real-world       │
 │  3. Visualize      An interactive metaphor you can touch       │
 │  4. See the code   Annotated example — every piece explained   │
 │  5. Do it          Live playground — edit & watch it update    │
 │  6. Avoid traps    Common mistakes (✗ vs ✓ side by side)       │
 │  7. Prove it       Quick quiz — explains every answer          │
 │  8. Reward         Mark complete → +XP → next step unlocks     │
 │     (Quest, the AI tutor, is present the entire time)          │
 └──────────────────────────────────────────────────────────────┘
```

## Progression model

- **Stages** are themed chapters (13 total). A stage **unlocks** when the previous one is fully complete (Stage 1 always open). This creates momentum and a sense of earned progress without hard-blocking the curious (the roadmap still *shows* everything).
- **Lessons** within a stage are completed via the "Mark complete" action and/or finishing the quiz.
- **XP** rewards every completion (+20). It's a dopamine nudge, not a gate.
- **Overall progress** (done/total + %) is always visible in the header and on the roadmap.
- **Summit:** completing all stages reaches "Compose Master" 🏆.

## "Where am I?" — the four orientation questions, always answered

| Question | Where it's answered |
|---|---|
| Where am I now? | Highlighted current node on the roadmap; breadcrumb on lesson pages. |
| What have I completed? | Green check nodes; per-stage `done/total`; header progress bar. |
| What comes next? | The next unlocked node; "Next →" button on every lesson. |
| How far from mastery? | Overall % bar; the summit flag at the end of the roadmap. |

## Difficulty ramp

The 13 stages are ordered so each builds only on prior knowledge:

`Fundamentals → Layouts → Modifiers → State → Lists → Navigation → Side Effects → Architecture → Networking → Local Storage → Animations → Performance → Advanced`

State (the classic beginner wall) arrives *after* the learner is comfortable creating and arranging UI, and is taught with the strongest metaphors and the most authored depth.

## Accessibility of understanding

- The **"Like I'm 10"** tab is a first-class citizen, not a gimmick — it's the safety net.
- The **AI mentor** can re-explain anything at any level on demand.
- **Visual metaphors** carry the conceptual load so text can stay short.
- **No dead ends:** if a learner is stuck, the mentor and the "like I'm 10" view are always within reach.
